# 확장앱 개발기획서(초안)
프로젝트명: `ThreadHook` (가칭)  
작성일: 2026-03-06

## 1) 프로젝트 개요
최근 이슈가 되는 블로그/뉴스 페이지를 탐색할 때, 현재 페이지의 핵심 정보를 자동 추출·요약하고 스레드(Threads) 형태로 즉시 공유 가능한 게시글 초안을 생성하는 브라우저 확장앱을 개발한다.

## 2) 목표
- 사용자가 읽는 페이지를 기반으로 30초 내 공유 가능한 정보글 생성
- 사실 기반 요약 + 출처 링크 자동 포함
- 톤/길이/해시태그를 채널 특성에 맞게 자동 최적화
- 반복 수작업(복사/요약/정리/작성) 최소화

## 3) 핵심 사용자
- 뉴스 큐레이터, 1인 미디어 운영자, 마케터, 커뮤니티 관리자
- “읽자마자 공유”가 필요한 정보 소비/생산형 사용자

## 4) 문제 정의
현재는 사용자가 직접 핵심 문장 선별, 요약, 제목 작성, 출처 정리, 해시태그 구성까지 수행해야 해 공유 속도와 품질이 불안정하다.

## 5) 주요 기능(필수)
1. **페이지 컨텍스트 추출**
- 현재 탭의 제목, 본문, 메타정보, 이미지, URL, 발행일 추출
- 광고/네비/댓글 등 노이즈 제거

2. **이슈성 판단(트렌드 신호)**
- 키워드 급증, 반복 언급, 최근성 기준으로 이슈 점수 산정
- “왜 지금 이슈인지” 한 줄 설명 생성

3. **스레드용 정보글 자동 생성**
- 포맷: 훅 문장 → 핵심 3~5포인트 → 인사이트 → 출처
- 길이 옵션(짧게/보통/길게), 톤 옵션(중립/전문/캐주얼)
- 해시태그 자동 추천

4. **인용·출처 신뢰성 처리**
- 원문 링크 자동 삽입
- 추정/의견 문장 라벨링
- 게시 전 사용자 편집 화면 제공

5. **원클릭 공유 연동**
- Threads 공유용 복사/전송 지원
- 멀티채널 확장 고려(X, LinkedIn 등은 후속)

## 6) 사용자 플로우
1. 사용자가 뉴스/블로그 페이지 방문
2. 확장앱 아이콘 클릭
3. “요약+스레드 생성” 실행
4. 생성 결과(제목/본문/해시태그/출처) 미리보기
5. 사용자 수정 후 게시 또는 복사

## 7) 기술 아키텍처(권장)
- **Frontend**: Chrome Extension (Manifest V3), React + TypeScript
- **Content Script**: DOM 파싱, 본문 추출(Readability 계열)
- **Background Service Worker**: 인증, API 호출, 상태관리
- **Backend API**: 요약/문체변환/랭킹 로직, 사용자 설정 저장
- **AI 모듈**: 요약, 제목 생성, 훅 문장, 해시태그 추천
- **DB**: 사용자 설정/히스토리/템플릿
- **Observability**: 로그, 실패율, 생성시간, 수정률 추적

## 8) 비기능 요구사항
- 생성 시간: 평균 5초 이내
- 가용성: 99.5%+
- 개인정보 최소수집(원문 전문 저장 기본 비활성)
- 한국어 우선, 다국어 확장 가능 구조

## 9) 정책/리스크
- 저작권: 원문 과다 복제 금지, 요약 중심 + 출처 명시
- 허위정보: 사실/추정 구분 표기, 사용자 최종 검수 필수
- 플랫폼 정책: Threads 자동화/게시 정책 준수 필요

## 10) MVP 범위(1차)
- 뉴스/블로그 본문 추출
- 스레드형 초안 자동 생성(3가지 톤)
- 출처 자동 삽입
- 클립보드 복사 + 간편 공유

## 11) 제외 범위(1차)
- 완전 자동 게시
- 실시간 대규모 트렌드 크롤링 대시보드
- 팀 협업 워크플로우

## 12) 일정(예시, 8주)
1. 1~2주: 요구사항 확정, UX 와이어, 기술검증
2. 3~4주: 확장앱 본체 + 페이지 추출기 개발
3. 5~6주: AI 생성 파이프라인 + 편집 UI
4. 7주: QA, 정책 점검, 성능 개선
5. 8주: 베타 배포, 사용자 피드백 반영

## 13) 성공 지표(KPI)
- 주간 활성 사용자(WAU)
- 생성 후 게시 전환율
- 평균 편집 시간/수정률
- 재사용률(주 2회 이상)
- 오류율(추출 실패/생성 실패)

## 14) 다음 단계
1. 타깃 사용자 5명 인터뷰로 필수 기능 우선순위 확정
2. 샘플 30개 기사로 요약 품질 기준(정확성/가독성) 수립
3. MVP 화면 설계(생성/편집/공유 3화면) 후 개발 착수
